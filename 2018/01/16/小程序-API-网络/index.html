小程序-API-网络

发表于 2018-01-16 | 分类于 小程序 ， 框架学习 ， API | 阅读次数: |

小程序提供的原生API可以方便的调起微信提供的能力

大部分接口都有以下共性

  • wx.on 开头的 API 是监听某个事件发生的API接口，接受一个 CALLBACK 函数作为参数。当该事件触发时，会调用 CALLBACK 函数。

  • 如未特殊约定，其他 API 接口都接受一个OBJECT作为参数。

  • OBJECT中可以指定success, fail, complete函数来接收接口调用结果。

    注：文件的临时路径，在小程序本次启动期间可以正常使用，如需持久保存，需在主动调用 wx.saveFile，才能在小程序下次启动时访问得到。

微信网络API规定

1.每个微信小程序需要事先在小程序后台设置一个通讯域名(只能用域名而且需要经过ICP备案)，小程序可以跟指定的域名与进行网络通信

2.超时时间都是60s,最大并发数是10个,网络请求的refererheader 不可设置。其格式固定为 https://servicewechat.com/{appid}/{version}/page-frame.html

3.服务器返回建议使用UTF-8编码

wx.request(OBJECT)

发情网络请求,通过method参数设置请求方式OPTIONS、GET、HEAD、POST、PUT、DELETE、TRACE、CONNECT

发送给服务器的数据是String

注意: url 中不能有端口

wx.uploadFile(OBJECT)

上传文件到服务器 , content-type 为 multipart/form-data

wx.downloadFile(OBJECT)

下载文件到本地

下载后的文件会存储到一个临时文件

wx.connectSocket(OBJECT)

创建一个 WebSocket 连接

wx.onSocketOpen(CALLBACK)

监听webSocket连接打开事件

wx.onSocketError(CALLBACK)

监听webSocket错误

wx.sendSocketMessage(OBJECT)

通过WebSocket连接发送数据,要先 wx.connectSocket，并在 wx.onSocketOpen回调之后才能发送。

wx.onSocketMessage(CALLBACK)

监听WebSocket接收到服务器的消息事件 (data)

wx.closeSocket(OBJECT)

关闭WebSocket连接

wx.onSocketClose(CALLBACK)

监听WebSocket关闭

除了上述的监听方法,connectSocket本身会返回一个webSocketTask任务 , 也可以发送消息监听事件等
